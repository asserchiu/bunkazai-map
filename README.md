# bunkazai-map

Interactive map of Japan's nationally designated cultural properties (国指定文化財等),
built from the Agency for Cultural Affairs database.

**Live data source:** https://kunishitei.bunka.go.jp/bsys/searchlist

## Usage

Open in a browser (requires a local HTTP server or GitHub Pages — `fetch()` won't work from a `file://` URL).

```sh
# Quick local server (Python)
python3 -m http.server 8080
# then open http://localhost:8080
```

| Page | Path | Description |
|---|---|---|
| Landing page | `index.html` | Project overview with link to map |
| Map | `map.html` | Interactive Leaflet map |

## Update Data

1. Download the latest combined CSV:

   ```sh
   curl --output download.csv https://kunishitei.bunka.go.jp/utile/csv-list
   ```

2. Rebuild the GeoJSON:

   ```sh
   python3 build.py
   ```

3. Commit and push:

   ```sh
   git add download.csv data/features.geojson
   git commit -m "chore: update cultural property data"
   git push
   ```

`data/features.geojson` must be committed so the static host can serve it.

## Tech Stack

| Component | Choice | Notes |
|---|---|---|
| Map renderer | [Leaflet](https://leafletjs.com/) v1.9 + [MarkerCluster](https://github.com/Leaflet/Leaflet.markercluster) | Simpler setup, no async style-load timing issues |
| Map tiles | CartoDB Voyager (default) | 7 providers selectable in UI; no API key required |
| Data format | GeoJSON | Generated by `build.py` from `download.csv` |
| Hosting | Any static file host | GitHub Pages, Cloudflare Pages, Netlify, etc. |

## Data Notes

- `download.csv` contains ~37,000 records across 19 categories.
- `build.py` exports ~28,000 records that have valid coordinates within Japan's bounds.
  Records with missing or out-of-range coordinates are skipped (see `DECISIONS.md`).
- `管理対象ID` is preserved as a string to retain leading zeros.
  Detail URL pattern: `https://kunishitei.bunka.go.jp/heritage/detail/{台帳ID}/{管理対象ID}`
